# Absorb: Auto-Generate Commit Messages

**Date:** 2026-03-16
**Status:** Draft

## Problem

`stack absorb` uses a hardcoded default commit message (`fixup: absorb changes from stack review`) when `-m` is not provided. The same generic message is used for every branch, even when absorb commits to multiple branches with different files.

Users shouldn't have to write commit messages for absorb — the tool has enough context (file names, branch descriptions) to generate meaningful per-branch messages automatically.

## Solution

Generate a descriptive commit message per-branch based on the files being absorbed. The `-m` flag remains as an explicit override.

### Message Generation Rules

1. **Single file:** `fixup: update <filename>` (basename only)
2. **Two files:** `fixup: update <file1>, <file2>`
3. **Three+ files:** `fixup: update <file1>, <file2> (+N more)`
4. **Deleted file(s):** Use "remove" instead of "update" when content is null
5. **Mix of updates and deletes:** Use "update" (more common action wins)

Examples:
```
fixup: update useColumnReorder.ts
fixup: update GroupedTable.tsx, DataTable.tsx
fixup: update schema.ts, types.ts (+3 more)
fixup: remove oldHelper.ts
```

### Behavior Change

| Before | After |
|--------|-------|
| Same message for all branches | Per-branch message based on files |
| `-m` required for meaningful messages | Auto-generated by default |
| `-m` overrides default | `-m` still overrides (applied to all branches) |

## Implementation

### Changes to `src/commands/absorb.ts`

**1. Add `generateCommitMessage` helper (top of file or inline):**

```typescript
function generateCommitMessage(
  files: string[],
  fileContents: Map<string, Buffer | null>,
): string {
  const basenames = files.map((f) => f.split('/').pop() || f);
  const allDeleted = files.every((f) => fileContents.get(f) === null);
  const verb = allDeleted ? 'remove' : 'update';

  if (basenames.length === 1) {
    return `fixup: ${verb} ${basenames[0]}`;
  }
  if (basenames.length === 2) {
    return `fixup: ${verb} ${basenames[0]}, ${basenames[1]}`;
  }
  return `fixup: ${verb} ${basenames[0]}, ${basenames[1]} (+${basenames.length - 2} more)`;
}
```
